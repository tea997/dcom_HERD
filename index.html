Executive Summary

The HARK (Hearing Aid Relay Kit) project successfully developed a specialized Android application that transforms a standard smartphone into a high-quality, low-latency, portable table microphone for hearing aid users. This solution directly addresses the significant barrier of cost and inconvenience associated with proprietary hardware accessories.

Project Description

Problem Statement

Individuals with hearing impairments frequently encounter difficulties in environments with high background noise. Proprietary "table microphones" or "remote microphones" cost several hundred dollars and add inconvenience to daily routines.

Solution

HARK provides an elegant solution by utilizing the powerful, ubiquitous hardware already owned by the userâ€”their Android smartphone. It leverages the ASHA protocol, a standard feature in modern Android and hearing aid devices.

Key Features

  • Real-time, low-latency audio streaming
  • Optimized Bluetooth ASHA support
  • Background operation as Foreground Service
  • Customizable settings with gain adjustment
  • Hearing aid priority compatibility

Technical Implementation

Core Android Technologies

Kotlin

Primary language for modern features, null safety, and conciseness.

Audio APIs

AudioRecord & AudioTrack for direct low-level audio hardware access.

Foreground Service

Maintains audio stream even when app is not in foreground.

ASHA Protocol

Routs audio to hearing aids via optimized Bluetooth Low Energy.

Architecture

The application employs the Model-View-ViewModel (MVVM) architectural pattern with Jetpack Compose UI framework and Material Design accessibility standards.

Detailed Workflow

Application Flow

1

Initialization Phase

  • MainActivity handles setup and permissions
  • UserPreferencesRepository loads settings from DataStore
  • Bluetooth and audio permissions requested
2

Streaming Activation

  • User initiates streaming via UI button
  • Intent starts AudioStreamingService
  • Service promoted to foreground
3

Core Execution

  • High-priority thread captures audio
  • Gain adjustment applied to buffer
  • Audio routed to hearing aids via ASHA
4

Termination

  • Resources cleaned up when stopped
  • Service terminates gracefully
  • WakeLock and audio objects released

Audio Pipeline

The audio pipeline is optimized for minimal latency with input capture at 48kHz, gain adjustment processing, and output using PERFORMANCE_MODE_LOW_LATENCY with ACCESSIBILITY usage flags.

Conclusion

The HARK project has successfully delivered a high-impact, technically sophisticated solution to a common problem faced by the hearing impaired community.
